## 0. 文档说明

本文档基于 iOS 端 PRD V1.2 衍生，面向 **Web 端 MVP** 编写。Web 版与 iOS 版共享同一套核心业务逻辑（算级、进贡、房规），但在交互载体、数据持久化、分享机制与部署方式上做了平台适配。

**产品 thesis（一句话）：** 为线下掼蛋玩家提供一个无需注册、打开即用的浏览器记分工具，用「四方阵」交互快速录入名次并自动推导升级与进贡关系。

### 0.1 已确认决策（V1.1）

| # | 议题 | 决策 |
|---|------|------|
| 1 | 部署 | GitHub 仓库 + GitHub Actions 自动部署至 GitHub Pages |
| 2 | 设计 | **Web 独立微调** — 不共用 iOS Figma；保留色彩语义，布局/导航按 Web 习惯优化 |
| 3 | 抗贡房规 | 4 项预设，国标合计双大王为默认（见 §2.1、附录 C） |
| 4 | 确认下局 | **弹窗二次确认**，取消滑动/长按方案 |
| 5 | 牌型校验 | **Web 与 iOS 同步首发**，纳入 Phase 1 MVP |
| 6 | 名次录入 | **Phase 1 用名单选择器**；四方阵动画降为 Phase 2 |
| 7 | 历史/分享 | **Phase 1 最小集**：历史列表 + 纯文本复制战报 |

## 1. 产品概述

### 1.1 背景与理念

掼蛋线下对局中，记分、算级、进贡/还贡规则常引发争议并分散注意力。iOS 版已验证「四方阵一键录入 + 动态进贡指引」的交互价值；Web 版解决 **非 iPhone 用户**（Android 手机、平板、桌面浏览器）及 **不愿安装 App** 用户的使用需求。

Web 版延续 iOS 核心定位：**本地优先、界面直观、零广告、零学习成本**。

### 1.2 产品定位

| 维度 | 说明 |
|------|------|
| 品类 | 线下掼蛋辅助工具（非在线棋牌） |
| 差异化 | 四方阵沉浸式交互、动态进贡箭头、本地房规、无账号 |
| 与 iOS 关系 | 功能对齐、逻辑一致；**不**做跨端云同步（V1） |
| 部署形态 | GitHub Pages 静态站点 + PWA，可「添加到主屏幕」 |

### 1.3 目标用户

| 用户类型 | 痛点 | Web 端价值 |
|----------|------|------------|
| 线下掼蛋爱好者 | 纸笔易错、历史难保存 | 自动算级、本地历史、一键复制战报 |
| 掼蛋新手 | 不熟悉进贡/抗贡规则 | 录入名次后动态箭头 + 文案指引 |
| 非 iPhone 用户 | 无法使用 iOS App | 浏览器即开即用 |
| 存在规则争议的玩家 | 逢人配牌型难判断 | 牌型校验器（MVP 同步首发） |

### 1.4 核心使用场景

| 场景 | 时机 | 行为 | 体验 | 结果 |
|------|------|------|------|------|
| 开局设置 | 四人落座后 | 输入姓名、设定级数与首局台主、可选房规 | 表单简洁，默认值合理 | 进入记分主控台 |
| 单局录入 | 一局结束 | 点击 4 位玩家，按点击顺序写入第 1–4 名 | 同屏展示名次回显，支持再次点击反选 | 触发进贡文字指引 |
| 结算下局 | 录入完成后 | 点击按钮 → **弹窗确认** | 弹窗展示本局摘要，防误触 | 自动算级，更新轮次横幅 |
| 上局进贡确认 | 下一局开始后（看牌后） | 查看上局进贡文案；可选勾选抗贡 | 抗贡二次确认防误触 | 锁定并确认上局进贡/抗贡结果 |
| 牌型校验 | 出牌争议时 | 切换校验 Tab，选牌验证 | 同步当前级牌 | 合法/非法即时反馈 |
| 分享战报 | 一局或整场结束 | 复制纯文本战报 | 剪贴板 + Toast 提示 | 粘贴至微信/备忘录 |
| 历史回顾 | 任意时刻 | 打开历史列表查看明细 | 按时间倒序卡片 | 可查看级数快照与名次 |

### 1.5 MVP 边界（明确不做）

- ❌ 用户注册 / 登录 / 云端账号
- ❌ 跨设备数据同步（iOS ↔ Web）
- ❌ 在线对战、匹配、聊天
- ❌ 广告与商业化
- ❌ 复杂房规编辑器（MVP 仅 2 项核心分歧：打 A + 抗贡）
- ❌ 牌型比大小（校验器仅判合法性，不比大小）
- ❌ 后端 API / 数据库服务
- ❌ 多语言（V1 仅简体中文）

| F-002 | 南北为我方（翠绿），东西为对方（珊瑚红） | P0 | Given 完成设置，When 查看主控台，Then 我方/对方颜色与 PRD 一致 |

| F-004 | 可设定首局台主（先出牌方） | P0 | Given 选择「南北」为台主，When 进入首局，Then 我方卡片高亮 |
| F-005 | 房规：打 A 是否必须双下 | P1 | Given 勾选「打 A 必须双下」，When 过 A 结算，Then 按房规判定 |
| F-006 | 房规：抗贡预设（4 选 1，默认国标合计双大王） | P1 | Given 选择预设，When 下一局确认区勾选抗贡，Then 确认文案与规则说明一致 |
| F-007 | 设置数据持久化至 localStorage | P0 | Given 用户完成设置并刷新页面，When 重新打开，Then 恢复上次对局上下文 |

**抗贡房规预设（F-006 最终列表）：**

| ID | 名称 | 规则说明 |
|----|------|----------|
| `combined_double_joker` | 国标：合计双大王 **(默认)** | 应进贡方合计 2 张大王 → 抗贡，头游领出 |
| `double_down_joker_split` | 双下：各一大王或一方双大王 | 双下局：两下游各 1 大王，或任一人 2 大王 → 抗贡 |
| `four_jokers` | 四张王牌 | 应进贡方合计 4 张王（两大+两小）→ 抗贡 |
| `disabled` | 关闭抗贡 | 不允许抗贡，必须进贡；主控台隐藏抗贡勾选 |

## 3. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 首屏 LCP < 2.5s（4G）；算级/进贡计算 < 16ms；动画 60fps |
| 数据 | 全部 localStorage；单用户无服务端；容量预估 < 1MB |
| 隐私 | 无追踪、无 Cookie 横幅（无第三方分析）；无个人数据上传 |

## 5. UI/UX 设计原则

### 5.1 Web 独立视觉微调（不共用 iOS Figma）

**继承自 iOS 的语义：**

| 元素 | 规范 |
|------|------|
| 我方 | 翠绿色 `#22C55E` 系 |
| 对方 | 珊瑚红 `#F87171` 系 |
| 台主/轮次 | 金色 `#EAB308` 高亮 |
| 进贡箭头 | 黄色光效 `#FACC15` |

**Web 独立调整：**

| 维度 | iOS | Web 微调 |
|------|-----|----------|
| 导航 | TabView / NavigationStack | 底部固定 Tab Bar + 顶栏标题（更适合浏览器） |
| 材质 | Liquid Glass 为主 | 轻量 `backdrop-blur` + 实色卡片 fallback，减少 GPU 开销 |
| 四方阵 | 全屏沉浸 | **Phase 2**；Phase 1 用名次选择器 |
| 进贡指引 | 箭头动画 | **Phase 1 纯文字**；箭头 Phase 2 |
| 弹窗 | sheet / alert | 居中 Modal + 遮罩，桌面/移动统一；下局确认、抗贡确认均用 Modal |
| 字体 | SF Pro | 系统字体栈 `-apple-system, "PingFang SC", "Noto Sans SC"` |
| 主题 | Light/Dark 跟随系统 | 同，`prefers-color-scheme` |
| 牌型键盘 | 原生 picker 风格 | 横向滚动扑克牌 Chip，桌面端支持键盘快捷键 |

### 5.2 Web 特有交互适配

| iOS 能力 | Web 替代 |
|----------|----------|
| ShareLink | Web Share API → `navigator.clipboard.writeText` 降级 |
| 触觉反馈 (Haptics) | `navigator.vibrate(50)`（仅支持的设备） |
| 下局确认 | **Modal 弹窗**（已确认，不用滑动/长按） |
| SwiftUI 动画 | CSS `@keyframes` + SVG 路径动画 |

### 5.3 四方阵布局

- **移动端**：正方形区域占屏宽 85–90%，四边圆形按钮直径 ≥ 56px（触控目标）
- **桌面端**：同比例缩放，最大 400×400px，居中；支持键盘 1–4 快捷录入（可选 P1）
- **进贡箭头**：SVG 路径动画，黄色光效 `#FACC15`

## 6. 技术架构

### 6.1 技术选型

| 层级 | 选型 | 理由 |
|------|------|------|
| 框架 | React 19 + TypeScript | 生态成熟、与 Roadmap 一致 |
| 构建 | Vite | 快速 HMR、静态部署友好 |
| 样式 | Tailwind CSS v4 | 快速实现响应式与主题 |
| 路由 | React Router v7 | 多页面 + 深链接；`basename` 适配 GitHub Pages |
| 状态 | Zustand 或 React Context | 轻量，无 Redux 过重 |
| 持久化 | localStorage + 自定义序列化 | 简单可靠；未来可升级 IndexedDB |
| 测试 | Vitest + Testing Library | 算级/进贡/牌型单元测试 |
| PWA | vite-plugin-pwa | 离线缓存、可安装 |
| 部署 | **GitHub Pages + GitHub Actions** | 零成本、与仓库同步 |

### 6.3 核心数据结构

**会话字段语义：**

## 附录 C：抗贡房规预设详解

### C.1 规则依据

| 来源 | 抗贡条件 |
|------|----------|
| 国标 / JJ 掼蛋 | 下游（或进贡方）持有 2 张大王 → 抗贡，头游领出 |
| 竞技规则 PDF | 双下：各抓 1 大王，或一方抓 2 大王 → 均不进贡 |
| 部分平台 | 进贡方合计 4 张王牌 → 抗贡 |
| 线下无抗贡玩法 | 必须进贡，不可豁免 |

### C.2 预设与 UI 行为

| 预设 ID | 设置页文案 | 抗贡确认弹窗 | 抗贡成功后 |
|---------|-----------|-------------|-----------|
| `combined_double_joker` | 国标：合计双大王 | 「确认进贡方合计抓到 2 张大王？」 | 抗贡！{头游} 领出 |
| `double_down_joker_split` | 双下：各一大王或一方双大王 | 「确认双下局满足抗贡条件？」 | 抗贡！{头游} 领出 |
| `four_jokers` | 四张王牌 | 「确认进贡方合计抓到全部 4 张王牌？」 | 抗贡！{头游} 领出 |
| `disabled` | 关闭抗贡 | （无，主控台不显示抗贡勾选） | — |

### C.3 实现位置

- 类型：`src/types/game.ts` → `HouseRules.antiTributePreset`
- 预设常量：`src/domain/houseRules.ts` → `ANTI_TRIBUTE_PRESETS`
- 默认：`combined_double_joker`

### C.4 领出规则（所有预设通用）

抗贡成功后，**上局头游**获得下一局首轮出牌权；不进贡、不还贡。
