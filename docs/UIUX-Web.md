## 0. 文档说明

本文档定义 **视觉语言、组件规格、页面线框、交互与动效**，供 Phase 1 正式实施对照。  
回答「长什么样、怎么点、各状态如何呈现」——不重复 PRD 功能定义与 SDD 技术实现。

**UX 目标（一句话）：** 牌局间隙 30 秒内完成名次录入，进贡关系一眼看懂，全程无需读说明书。

**Phase 1 交互策略：** 用 **名次选择器**（第 1–4 名各选玩家）替代四方阵；进贡用 **纯文字** 展示。四方阵 + 箭头动画延至 Phase 2。

**设计原则：**

1. **内容优先** — 级数、轮次、四方阵占据主视觉；装饰克制
2. **语义色彩** — 绿=我方、红=对方、金=台主，全局一致
3. **拇指友好** — 主操作在屏幕中下部；触控目标 ≥ 44px
4. **防误触** — 抗贡、下局、删除均二次确认
5. **Inclusive** — 支持 Dark Mode、减少动效、键盘可操作

## 1. 设计 Token

### 1.1 色彩

| Token | Light | Dark | 用途 |
|-------|-------|------|------|
| `--color-our` | `#22C55E` | `#22C55E` | 我方队、主 Tab 激活、主按钮 |
| `--color-our-muted` | `#16A34A` | `#16A34A` | 我方 hover、次级强调 |
| `--color-opponent` | `#F87171` | `#F87171` | 对方队、警告性强调 |
| `--color-opponent-muted` | `#EF4444` | `#EF4444` | 对方 hover |
| `--color-dealer` | `#EAB308` | `#EAB308` | 台主、轮次横幅、皇冠 |
| `--color-tribute` | `#FACC15` | `#FACC15` | 进贡箭头、进贡文案高亮 |
| `bg-page` | `neutral-50` | `neutral-950` | 页面背景 |
| `bg-surface` | `white/80` | `neutral-900/80` | 卡片、Tab Bar |
| `border-default` | `neutral-200` | `neutral-800` | 分割线、卡片边框 |
| `text-primary` | `neutral-900` | `neutral-100` | 正文 |
| `text-secondary` | `neutral-500` | `neutral-400` | 辅助说明 |
| `text-error` | `red-600` | `red-400` | 非法牌型、错误提示 |
| `text-success` | `our` | `our` | 校验合法 |

**对比度：** 正文与背景 ≥ 4.5:1（WCAG AA）；金色横幅上文字用 `amber-900` / `amber-100` 保证可读。

### 1.2 字体

| Token | 值 |
|-------|-----|
| `--font-sans` | `-apple-system, "PingFang SC", "Noto Sans SC", sans-serif` |

| 样式 | 大小 | 字重 | 行高 | 用途 |
|------|------|------|------|------|
| `display` | 30px (1.875rem) | 700 | 1.2 | 级数大数字 |
| `title-lg` | 20px (1.25rem) | 600 | 1.3 | 页面标题 |
| `title-md` | 16px (1rem) | 600 | 1.4 | 卡片标题、Modal 标题 |
| `body` | 14px (0.875rem) | 400 | 1.5 | 正文、说明 |
| `caption` | 12px (0.75rem) | 400 | 1.4 | 提示、时间戳 |
| `label` | 14px | 500 | 1 | 按钮、Tab |

**可读性增强（老年用户）：**
- 默认基线字体放大到 110%
- 提供全局字体缩放控件：`A- / A+ / 重置`（100%–150%，步进 10%，localStorage 持久化）

### 1.3 间距与圆角（8px 网格）

| Token | 值 | 用途 |
|-------|-----|------|
| `space-1` | 4px | 紧凑内边距 |
| `space-2` | 8px | Chip 间距 |
| `space-3` | 12px | 表单项间距 |
| `space-4` | 16px | 页面水平 padding |
| `space-6` | 24px | 区块间距 |
| `radius-sm` | 8px | 输入框、Chip |
| `radius-md` | 12px | 按钮 |
| `radius-lg` | 16px | 卡片 |
| `radius-xl` | 24px | 四方阵容器、Modal |
| `radius-full` | 9999px | 玩家圆形节点 |

### 1.4 阴影与材质

| 场景 | 样式 |
|------|------|
| 卡片默认 | `border` + 无阴影或 `shadow-sm` |
| 台主卡片高亮 | `border-2` + `shadow-md` + 对应队色 `ring-2 ring-{color}/30` |
| Tab Bar | `border-t` + `bg-surface backdrop-blur-md` |
| Modal | `shadow-xl` + `bg-white dark:bg-neutral-900` |
| 毛玻璃 fallback | `@supports not (backdrop-filter)` → 实色 `bg-white` / `bg-neutral-900` |

### 1.5 动效

| Token | 值 |
|-------|-----|
| `duration-fast` | 150ms |
| `duration-normal` | 250ms |
| `duration-slow` | 400ms |
| `ease-out` | `cubic-bezier(0, 0, 0.2, 1)` |
| `ease-spring` | `cubic-bezier(0.34, 1.56, 0.64, 1)` |

| 触发 | 动画 | 时长 | Reduce Motion |
|------|------|------|---------------|
| Tab 切换 | 颜色 crossfade | 150ms | 即时 |
| 名次录入 | 节点 scale 1→1.08→1 + 数字淡入 | 250ms | 仅数字淡入 |
| 进贡箭头 | SVG stroke-dashoffset | 400ms | 静态箭头 |
| Modal 打开 | 遮罩 fade + 内容 scale 0.95→1 | 250ms | 仅 fade |
| 校验失败 | 水平 shake + 红框 | 300ms | 仅红框 |

## 4. 页面规格

### 4.1 设置页 `/setup`

**目标：** 30 秒内完成四人绑定与房规，进入主控台。

```
┌─────────────────────────────────────┐
│ 开局设置                             │
│ 绑定座位、级数与房规                   │
├─────────────────────────────────────┤
│ 东  [___________]  ← 对方 珊瑚色标    │
│ 南  [___________]  ← 我方             │
│ 西  [___________]  ← 对方             │
│ 北  [___________]  ← 我方             │
├─────────────────────────────────────┤
│ 初始级数                             │
│  南北 [2 ▼]    东西 [2 ▼]            │
│ 首局台主  ○ 南北  ○ 东西             │
├─────────────────────────────────────┤
│ ▼ 房规（折叠）                        │
│  ☑ 打 A 必须双下                      │
│  抗贡规则  [国标：合计双大王 ▼]        │
├─────────────────────────────────────┤
│ ┌─────────────────────────────────┐ │
│ │      进入记分主控台               │ │  ← primary 全宽
│ └─────────────────────────────────┘ │
│  新对局（有 session 时，destructive）  │
└─────────────────────────────────────┘
```

| 状态 | 表现 |
|------|------|
| 首次进入 | 默认级数 2，台主南北，房规默认 |
| 有进行中 session | 表单预填；底部「新对局」需 ConfirmDialog |
| 校验 | 级数 2–14；姓名可空 |

### 4.2 记分主控台 `/game`（核心）

**Phase 状态机：** `pending-review?` → `ranking` → `confirm`

#### 4.2.1 ranking 阶段

```
┌─────────────────────────────────────┐
│ 第 3 局                              │
├─────────────────────────────────────┤
│ 👑 当前轮次：南北队打 8  ← playingTeam│
├─────────────────────────────────────┤
│ [我方 8·先出牌] [对方 6] ← dealer    │
├─────────────────────────────────────┤
│ 录入名次（RankPicker）               │
│  点击 4 位玩家按顺序录入              │
│  [撤销上一步] [清空重选]             │
└─────────────────────────────────────┘
```

| 交互 | 响应 |
|------|------|
| 点击未选玩家 | 写入下一名次（1→4） |
| 再点已选玩家 | 反选并紧凑前移 |
| 撤销重选 | 清空 4 名，回 ranking 初态 |
| 有上局待确认 | ranking 区禁用，提示先确认上局进贡 |

#### 4.2.2 上局进贡确认阶段（下一局开始后）

```
├─────────────────────────────────────┤
│ 上局进贡确认（第 N 局）               │
│  · 赵六 → 张三：进贡最大牌            │
│  ☐ 抗贡（看牌后确认）                 │
│ [确认上局进贡，开始录入本局名次]       │
├─────────────────────────────────────┤
│ [ 确认并进入下局 ]（未确认上局时禁用） │
└─────────────────────────────────────┘
```

| 交互 | 响应 |
|------|------|
| 勾选抗贡 | ConfirmDialog（文案随房规） |
| 确认上局进贡 | 写入上局 `isAntiTribute` 后解锁 ranking |
| 点击确认下局 | 打开 `RoundConfirmModal`（仅展示进贡预览） |

```mermaid
    G --> H[进贡文字预览]
    H --> I[点击确认下局]
    I --> J[RoundConfirmModal]
    J -->|取消| G
    J -->|确认| K[结算 + 下一局]
    K --> D
```

| 项 | 要求 |
|----|------|

| 字体缩放 | 提供 `A- / A+ / 重置`，支持 1 米观看场景下快速放大 |

## 8. 文案规范

| 类型 | 风格 | 示例 |
|------|------|------|
| 标题 | 简短名词 | 「开局设置」「历史记录」 |
| 说明 | 一句操作指引 | 「请按出完顺序点击 4 位玩家」 |
| 确认 | 直接问句 | 「确认进贡方合计抓到 2 张大王？」 |
| 成功 | 叹号 + 结果 | 「抗贡！张三 领出」 |
| 错误 | 无错码 | 「此排名不符合规则」 |
| 空态 | 中性 + 引导 | 「暂无历史记录，去记一局吧」 |

**禁止：** 技术术语（state machine、localStorage）；冗长规则全文（房规用折叠 + 一句摘要）。
